The fastest path to a working AdGem integration. This guide takes you from a fresh AdGem account to a live Offer Wall that pays out real rewards to your players — with server-to-server postback verification — in about five minutes.

Quickstart: 5-Minute Postback

What you'll build

A working Offer Wall embedded in your site, wired to a postback endpoint that receives (and verifies) AdGem's conversion callbacks. Everything you need to start rewarding players on day one.

Prerequisites​

Before You Begin

Account Requirements:

  • You have an active AdGem Account
  • You have added your app to your Account

App Approval:

  • Your app has been approved in the AdGem system
Important

By default, your app will not have access to AdGem's offers until you complete the initial integration steps and your app is approved. Contact your dedicated Publisher Support Advocate with any questions about the approval process.

:::

Platform Requirements​

Web Offerwall Requirements
  • Modern web browser with JavaScript enabled
  • HTTPS-enabled website (recommended for production)
  • Server-side postback endpoint

You'll also need:

  • A server endpoint you can POST/GET to (for the postback — any https:// URL your team controls)
  • Five uninterrupted minutes

Step 1 — Create a Web property​

In the AdGem Publisher Dashboard:

  1. Click Properties & Apps in the left nav.
  2. Click the New Property button.
  3. Select Desktop/Web from the Platform dropdown.
  4. Name your property and submit the form.
note

You need a distinct property for every website URL you integrate. Stash your new Property ID — you'll need it in the next step.

Step 2 — Drop the Offer Wall into your site​

On your new property's settings page, open the Offerwall tab. Copy the Web Offerwall Direct Link and append a playerid query param for your logged-in user:

<a href="https://api.adgem.com/v1/wall?appid=YOUR_APP_ID&playerid=USER_ID" target="_blank">
  Earn Rewards
</a>

Prefer an embedded experience? Use the Full Screen iFrame Code instead:

<iframe
  src="https://api.adgem.com/v1/wall?appid=YOUR_APP_ID&playerid=USER_ID"
  width="100%"
  height="600"
  frameBorder="0">
</iframe>

Load the page. If you see the Offer Wall, you're halfway home.

Step 3 — Configure your postback​

Back in the dashboard, scroll down to Postback Options on the same property page. Pick Server Postback.

  1. Copy the Postback Key that appears. This is shown once — save it somewhere secure (a secrets manager, .env, or your deploy pipeline).

  2. Paste your public endpoint URL into the Postback URL field. Use AdGem's macros to populate the parameters — at minimum:

    https://your-server.example.com/postbacks/adgem?player_id={playerid}&amount={amount}&conversion_id={conversion_id}&hash={hash}

  3. Save.

Postback macros

AdGem replaces {playerid}, {amount}, {conversion_id}, {hash}, and more before calling your URL. See Postback Setup for the full macro list.

Step 4 — Verify a postback​

When a player completes an offer, AdGem calls your endpoint. Verify the hash parameter before crediting the player:

const crypto = require('crypto');
const express = require('express');

const app = express();

app.get('/postbacks/adgem', (req, res) => {
  const { player_id, amount, conversion_id, hash } = req.query;

  const expected = crypto
    .createHash('md5')
    .update(`${conversion_id}${process.env.ADGEM_POSTBACK_KEY}`)
    .digest('hex');

  if (hash !== expected) {
    return res.status(401).send('Invalid signature');
  }

  // Credit the player
  rewardPlayer(player_id, Number(amount));
  res.status(200).send('OK');
});

app.listen(3000);

Trigger a test conversion from the AdGem dashboard (your Publisher Support Advocate can flip this on for you). You should see the GET hit your server, verify cleanly, and reward the player.

That's it. You have a working Offer Wall and an authenticated postback pipeline.
